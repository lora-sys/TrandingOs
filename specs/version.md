# Trading Pi Vision

Trading Pi is not a trading bot, signal caller, or multi-agent framework.

Trading Pi is a personal trading operating system that helps a user learn, research, plan, execute safely, review, evolve, and eventually form their own alpha.

## Core Mission

Trading Pi helps the user move through this path:

Beginner -> market observer -> researcher -> planner -> executor -> reviewer -> optimizer -> independent trader.

The goal is not to replace the user. The goal is to help the user grow.

## Non-Negotiable Principle

Trading Pi is Single Agent First.

There is only one core agent:

`Trading Pi Agent`

The system must not introduce:

- Research Agent
- Risk Agent
- Portfolio Agent
- Execution Agent

All capabilities must flow through:

`Trading Pi Agent -> Workflow -> Skills -> Artifacts`

## Product Positioning

Trading Pi combines the feel of:

- Claude
- Cursor
- TradingView
- Journal
- Coach

It is simultaneously:

- AI assistant
- Trading coach
- Research system
- Review system
- Strategy evolution system
- Personal knowledge system

## Five Core Capabilities

### Research

Research includes news, macro, onchain, market data, files, and browser context.

Primary output: `Research Artifact`.

### Planning

Planning includes bull case, bear case, technical case, risk case, debate, and final plan.

Primary output: `Trade Plan`.

### Execution

Execution supports mock, paper, and live guarded modes.

Default mode: `Paper First`.

Every dangerous operation must require approval.

### Review

Review includes daily, weekly, monthly, and discipline review.

Primary output: `Review Artifact`.

### Evolution

Evolution includes pattern detection, strategy optimization, skill evolution, and workflow evolution.

Primary output: `Evolution Proposal`.

## Product Rules

- Artifact First: important results become traceable artifacts.
- Preview First: artifacts support Markdown, HTML, PDF, and screenshot preview.
- Workspace First: workspace means context + artifacts + memory + workflows.
- Memory First: the system remembers conversations, trades, reviews, patterns, and skill records.
- Skill First: capability comes from Skill Registry, not hardcoded feature branches.
- Workflow First: complex business logic belongs in workflows, not new agents.
- Browser + Search: real-world awareness comes from AIO Sandbox and Search Hub.
- MCP: MCP is the extension layer for registry, discovery, health, and marketplace.
- Marketplace: skills, workflows, templates, and MCP integrations become an ecosystem.
- Journal: growth is recorded through reasons, screenshots, emotions, discipline violations, and lessons.

## Trading Modes

- Mock
- Paper
- Live Guarded

Trading Pi must never default to live trading.

## Final Goal

Trading Pi does not optimize for taking control away from the user.

Trading Pi optimizes for helping the user become a long-lived, disciplined, evolving independent trader with personal alpha.

If future development creates a conflict, prefer helping the user grow over operating for the user.
