# Research + Airdrop Learning Spec v4.1

## 1. Purpose

Trading Pi should teach beginners not only trading but also Web3 research and airdrop participation.

The system must not automatically operate user wallets.

It can:

- search opportunities
- explain tasks
- verify official sources
- detect scam risk
- create step-by-step guides
- track learning progress

## 2. Airdrop Workflow

```txt
User asks about airdrop
  ↓
Research Hub searches official sources
  ↓
Browser Layer verifies website
  ↓
Scam Check Skill evaluates risk
  ↓
Eligibility Skill creates checklist
  ↓
Artifact Engine creates Airdrop Guide
  ↓
User manually executes if safe
```

## 3. Airdrop Skills

- airdrop.search_opportunities
- airdrop.verify_official_source
- airdrop.scam_check
- airdrop.eligibility_check
- airdrop.task_breakdown
- airdrop.create_guide
- airdrop.track_progress

## 4. Safety Rules

- Never ask for seed phrase.
- Never sign transactions automatically.
- Never connect wallet automatically.
- Never approve token spending automatically.
- Always show phishing warning.
- Always use Browser Preview for official pages.
- Always create artifact guide before user acts.

## 5. Artifacts

- Airdrop Opportunity Report
- Eligibility Checklist
- Scam Risk Report
- Step-by-Step Task Guide
- Progress Tracker
